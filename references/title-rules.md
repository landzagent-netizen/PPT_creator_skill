# 乔布斯式标题生成规则

## 核心原则

- **一句话原则**：每个标题必须在一口气内说完（≤ 12 字中文 / ≤ 8 词英文）
- **对比鲜明**：用反差制造记忆点
- **具象化**：用具体事物解释抽象概念
- **痛点直击**：问题式标题引发共鸣

## 标题类型模板

| 类型 | 公式 | 示例 |
|------|------|------|
| 重新定义型 | [具体事物] + [颠覆性动作] | "1000 首歌，装进口袋" |
| 数字冲击型 | [震撼数字] + [简短结论] | "10 亿次，每一天" |
| 极简宣言型 | [一个形容词/短语] + [产品/概念] | "轻，超乎想象" |
| 问题型 | [如果/为什么/凭什么] + [核心矛盾] | "凭什么你要等 3 分钟？" |
| 对比颠覆型 | [旧认知] → [新事实] | "3 天 → 3 分钟" |
| 比喻型 | [抽象概念] + "就是" + [具体事物] | "API 就是服务员" |
| 痛点直击型 | [听众的痛] + [暗示解法] | "每次部署都像拆炸弹？" |

**场景偏好**：
- launch：偏重重新定义型、极简宣言型，追求震撼
- edu：偏重问题型、比喻型，追求引导思考

## 标题生成四步法

**Step 1：提取核心概念**
- 从章节口播稿中找到这一章的唯一核心信息
- 问自己：如果听众只记住一句话，是哪句？
- 一个章节 = 一页幻灯片 = 一个核心概念，不允许塞两个

**Step 2：寻找具象化比喻对象**
- 为抽象概念找一个日常生活中的具体事物
  - "高并发" → "春运抢票"
  - "端到端加密" → "只有你和对方有钥匙的保险箱"
  - "微服务" → "乐高积木"
- 优先选择听众 5 秒内能理解的比喻
- 找不到好比喻时，用数字对比代替

**Step 3：压缩文字**
- 砍掉所有形容词和副词
- 能用数字就不用文字
- 能用符号（→、×、=）就不用连接词
- 反复删减直到 ≤ 12 字

**Step 4：质量检查**
- ≤ 12 字？超出必须重写
- 一口气能说完？不能有停顿感
- 引发好奇？读完会不会想看内容
- 脱离上下文能独立理解？不依赖前后页
- 有画面感？脑中能形成图像
- 避免术语和缩写？非技术听众也能懂

# 配色系统

## 主色规则（核心原则）

**全篇背景和视觉元素只用 2-3 种主色**，贯穿全部页面。

| 元素类型 | 颜色规则 |
|---------|---------|
| 背景色 | 全篇统一，1 种 |
| 视觉元素色（装饰图形、图标、形状） | 全篇统一，最多 2 种辅助色 |
| 标题文字色 | 可根据页面内容变化，突出核心 |
| 正文文字色 | 浅灰色或白色，确保可读性 |

**禁止**：
- 每个页面随意切换背景色
- 同一概念在不同页面用不同颜色
- 彩虹效果（每页颜色不超过 2-3 种）

## 语义色表（用于标题和正文着色）

| 概念类型 | 颜色 | 示例文本 |
|---------|------|---------|
| 工具/技术 | Cyan / Blue / Purple | "SDK"、"API"、"给 AI 的工具箱" |
| 痛点/问题 | Red / Pink | "不可控"、"效率低下"、"每次都崩" |
| 解决方案 | Green / Teal | "一键生成"、"自动化"、"3 分钟搞定" |
| 数据/指标 | Orange / Amber | "提升 80%"、"10 亿次"、"99.9%" |
| 核心概念 | White（深底）/ Black（浅底） | 标题主文字 |
| 引用/旁注 | Gray / Muted | 补充说明、引述 |
| 行动号召 | 品牌主色 / 高饱和强调色 | "现在就试试"、"扫码体验" |
| 对比-旧 | Gray / Desaturated | "传统方式"、"过去" |
| 对比-新 | 亮色 / Saturated | "新方案"、"现在" |

## 着色规则

- 每页幻灯片最多 2-3 种颜色（包括文字色）
- 标题用核心概念色，关键词用对应语义色高亮
- 同一概念类型在整个 PPT 中颜色保持一致
- 背景色和视觉元素色全篇统一，不得随意更换
